## Очерёдность (от безопасного к рискованному)

### 🟢 Шаг 1 — On-chain демо на live фронте (15 мин) — делай ПЕРВЫМ

Это даст тебе финальное недостающее поле в `BUILDER-SUBMISSION.md`.

1. Открой https://fhevm-oracle-frontend.vercel.app
2. MetaMask → Sepolia (на коше должно быть >0.05 SepETH; если мало — кран https://sepoliafaucet.com)
3. Connect → дефолтные значения `amount=12345`, `secret=0xc0ffee` оставь, либо поменяй на свои — оба поля обязательны → reveal in 60 sec → Lock → подтверди в MetaMask
4. Дождись 60 сек, нажми **Trigger reveal** → подтверди в MetaMask
5. После триггера в UI появится кнопка **Fulfill** — жми её, подтверди → cleartexts появятся в карточке vault
6. Открой свой адрес в Sepolia Etherscan, найди три транзакции к контракту `0x256e89...A49b`: `lock`, `triggerReveal`, `fulfillReveal`
7. Скопируй URL **`fulfillReveal` транзакции** (она самая показательная — там KMS-подписанный cleartext приземляется on-chain)
8. Открой `BUILDER-SUBMISSION.md` → найди поле `Sepolia transaction (lock + reveal demo)` → впиши URL → сохрани → коммит:

```
git add BUILDER-SUBMISSION.md
git commit -m "docs: fill Sepolia tx URL in BUILDER submission"
git push
```

**Если что-то падает** — дай мне в чат точную ошибку, разберёмся.
